# TradeSim: Algorithmic Trading Backtester

**TradeSim** — це високопродуктивна система для розробки, тестування та аналізу алгоритмічних торгових стратегій на історичних ринкових даних.

Проект дозволяє трейдерам та розробникам перевіряти гіпотези (бектестинг) без фінансових ризиків, використовуючи реальну історію котирувань криптовалют та акцій.

---

## Основний функціонал

* **Симуляція торгівлі:** Швидкий "прогон" стратегій на історичних даних (Historical Data Backtesting).
* **Гнучкий конструктор стратегій:** Можливість створювати власні торгові алгоритми на TypeScript, використовуючи готовий API для індикаторів (RSI, SMA, MACD тощо).
* **Аналітика та метрики:** Розрахунок ключових показників ефективності: PnL (прибуток/збиток), Win Rate, Max Drawdown (максимальна просадка), Sharpe Ratio.
* **Висока продуктивність:** Використання **Node.js Worker Threads** для паралельного тестування стратегій на десятках активів одночасно.
* **Потокова обробка даних:** Ефективне зчитування великих масивів фінансових даних (CSV/JSON) без перевантаження оперативної пам'яті (Streams).

## Технологічний стек

* **Core:** [TypeScript](https://www.typescriptlang.org/) (Strict Mode) — для надійної типізації фінансових даних.
* **Runtime:** [Node.js](https://nodejs.org/) — серверне середовище виконання.
* **Architecture:** SOLID принципи, об'єктно-орієнтоване проектування (OOP), патерни GoF (Strategy, Factory, Observer, Adapter та ін.).
* **Performance:** Node.js Worker Threads (Multithreading) для важких обчислень.

## Архітектура проекту

Система побудована на модульній архітектурі, що дозволяє легко розширювати функціонал:
* **Data Layer:** Адаптери для різних джерел даних (Binance API, CSV файли, Yahoo Finance).
* **Strategy Layer:** Абстракції для реалізації торгової логіки.
* **Execution Engine:** Ядро симуляції, що відтворює ринкові умови (спреди, комісії).
